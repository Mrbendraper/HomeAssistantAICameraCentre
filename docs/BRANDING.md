# Branding

The brand assets live in two places:

- [`custom_components/ai_camera_centre/brand/`](../custom_components/ai_camera_centre/brand)
  — `icon.png` (256×256), `icon@2x.png` (512×512), `logo.png` (510×128)
  and `logo@2x.png` (1020×256). These ship with the integration itself.
- [`assets/logo-wide.png`](../assets/logo-wide.png) — larger wordmark used
  at the top of the README.

All of them are generated by [`scripts/generate_brand.py`](../scripts/generate_brand.py)
(requires Pillow; uses Segoe UI Bold, so run it on Windows or adjust
`FONT_PATH`). Edit the script and re-run it to change the design — don't
edit the PNGs directly.

## How the icon reaches the Home Assistant UI

Since **Home Assistant 2026.3**, custom integrations bundle their brand
images directly: a `brand/` folder inside the integration
(`custom_components/ai_camera_centre/brand/`) containing `icon.png` and
`logo.png` (plus optional `@2x` high-DPI and `dark_icon.png` /
`dark_logo.png` dark-theme variants). The frontend loads them through the
brands proxy API (`/api/brands/integration/ai_camera_centre/...`), and
locally bundled images automatically take priority over the central
brands CDN — no configuration and, crucially, **no PR to
home-assistant/brands** is needed any more (submissions of
integration-bundled brands there are now rejected).

On Home Assistant versions older than 2026.3 the folder is simply
ignored and the default placeholder icon shows; functionality is
unaffected.

If the icon doesn't appear after updating, it's almost always the
frontend cache — hard-refresh the browser (Ctrl+F5) or reset the
companion app's frontend cache.

## Other places worth branding (manual, one-time)

- **GitHub social preview**: repo → Settings → General → Social preview →
  upload `assets/logo-wide.png` (or a 1280×640 variant) — used when the
  repo is shared/linked, including from HACS.
- **HACS**: renders the README (including the header logo) on the repo
  page.

## Possible future addition

Dark-theme variants (`dark_icon.png`, `dark_logo.png`) — the current
white-tile design works on dark backgrounds, but a transparent-background
variant could look more native. Generate via a second palette in
`generate_brand.py` if wanted.
