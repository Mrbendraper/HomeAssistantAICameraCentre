# Branding

The brand assets live in two places:

- [`brand/ai_camera_centre/`](../brand/ai_camera_centre) — `icon.png`
  (256×256), `icon@2x.png` (512×512), `logo.png` (510×128) and
  `logo@2x.png` (1020×256), sized to pass the
  [home-assistant/brands](https://github.com/home-assistant/brands) CI
  checks.
- [`assets/logo-wide.png`](../assets/logo-wide.png) — larger wordmark used
  at the top of the README.

All of them are generated by [`scripts/generate_brand.py`](../scripts/generate_brand.py)
(requires Pillow; uses Segoe UI Bold, so run it on Windows or adjust
`FONT_PATH`). Edit the script and re-run it to change the design — don't
edit the PNGs directly.

## Getting the icon to show inside Home Assistant

Home Assistant does **not** read icons from the integration folder — every
integration's icon and logo come from the central
[home-assistant/brands](https://github.com/home-assistant/brands)
repository, served via `brands.home-assistant.io`. Until the assets are
merged there, the integration shows a default placeholder in Settings →
Devices & Services (functionality is unaffected).

One-time submission:

1. Fork `https://github.com/home-assistant/brands`
2. Create the folder `custom_integrations/ai_camera_centre/`
3. Copy the four files from this repo's `brand/ai_camera_centre/` into it
4. Open a PR titled "Add ai_camera_centre brand" — CI validates the sizes
   automatically; once merged, icons appear in HA (frontend caches may take
   a day, or a browser hard-refresh)

## Other places worth branding (manual, one-time)

- **GitHub social preview**: repo → Settings → General → Social preview →
  upload `assets/logo-wide.png` (or a 1280×640 variant) — used when the
  repo is shared/linked, including from HACS.
- **HACS**: shows the brands icon once the brands PR is merged, and renders
  the README (including the header logo) on the repo page.
